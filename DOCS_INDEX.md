## 使い分けの早見

- **Claude Codeに実装を頼む** → CLAUDE.md が自動で効く（規約）。
- **新しいAIチャットで相談を始める** → HANDOFF.md を貼る。
- **久しぶりに自分で触る／本番が変** → README.md の早見表。
- **そもそもどれ？** → このファイル。

## 整合性ルール（重要）

1. コードに構造変更（タブ/テーブル列/認証/コマンド/スコープ）を入れたら、**同じコミットで関連資料を更新**。
2. CLAUDE.md と HANDOFF.md は実態とズレたら価値を失う。実装後に必ず差分反映。
3. 資料を更新したら、このファイルの「最終更新」を直す。
4. コードを正とし、資料を合わせる。資料に合わせてコードを歪めない。
